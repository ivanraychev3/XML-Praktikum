1. Mancala.dtd --> NICHT AKTUELL

1.1. Header

Ein Mancala Spiel besitzt mehrere Attribute, die das Spiel dokumentieren.

<!ELEMENT game (finished,gameID,curplayer,slot*,player*)>

Dabei steht finished für die Beendigung des Spiels.
Mit gameID wird, wie der Name schon sagt, eine ID festgelegt, sodass mehrere Spiele gleichzeitig gespielt werden können, ohne das die Eingaben vertauscht werden.
curplayer signalisiert den aktuellen Spieler.
Zudem hat ein Spiel

2. MancalaGameState.xml

2.1. Header

<game gameID = "0" lastCount = "0" clickedHouseID = "0"> 
<finished>0</finished>
<curplayer>1</curplayer>

Der Header definiert verschiedene Spielobjekte wie die gameID, wodurch mehrere Spiele gleichzeitig gespielt werden können ohne das die Eingaben vertauscht werden.
Wobei das Spiel mit der ID = 0 das Default Spiel ist und als Vorlage für alle anderen Spiele gilt. 
In lastCount <text> wird die Anzahl des zuletzt geklickten Spielfeldes gespeichert um... Oder alle bisher gesammelten Steine.</text>
clickedHouseID verweist auf die ID des zuletzt geklickten Store bzw. 0 wenn noch nichts geklickt wurde.

3.1.1. clickhouse()

function clickhouse(ID)
{
	var gameIDlocal = '<xsl:value-of select="/*/@gameID"/>';
	var loc= window.location;
	window.location=("http://localhost:8984/clicked?ID="+ID+"&amp;gameID="+gameIDlocal);
}
